# Design Context — Resume Matcher

This file is the source of truth for design decisions. All `impeccable:*` skills (audit, critique, polish, animate, etc.) read this before making any visual changes.

> Last updated: 2026-07-15 — rebranded to the **Dark Fintech Design System** (warm-sunset palette). The prior light Swiss International Style was superseded; this file now reflects the dark theme that *is* the brand.

---

## Users

Resume Matcher serves three overlapping audiences, all of whom land on the same UI:

- **Anxious job seekers** — individuals tailoring resumes during active job hunts, often in the evening or late at night. They are the largest group. They want fast, calming, low-friction help — not AI hype, not clever features they have to learn. The interface should reduce their stress, not amplify it.
- **Tech-savvy DIY users** — developers, designers, and technical PMs who run things locally with Ollama. They notice and reward craft. They will silently judge a generic SaaS aesthetic and respect a distinctive one. They forgive complexity if it feels intentional.
- **Students and early-career applicants** — first-time job seekers who need confidence and clarity. The interface should feel encouraging without being patronizing or playful. It should make them feel like adults using a serious tool.

**Common job-to-be-done**: take an existing resume + a job description and produce a tailored resume + cover letter that the user can ship as a PDF. Speed and trustworthiness matter more than power-user features.

**Use context**: usually a desktop or laptop browser. Often used in 30–60 minute focused sessions. Frequently in the evening. Rarely on mobile (resume editing is a desktop task), but the marketing/landing surfaces still need to read on mobile.

## Brand Personality

Three words: **Confident · Honest · Crafted**

- **Confident** — quiet confidence, not loud. The interface doesn't need to shout that it's powerful. A confident UI does its job and trusts the user to notice the care.
- **Honest** — no inflated copy, no AI hype, no decorative ornament that doesn't earn its place. Status messages are direct. Errors say what's wrong. Buttons say what they do.
- **Crafted** — every pixel feels intentional. Typography, spacing, alignment, and color choices all reward closer inspection. The kind of interface that makes a designer say "someone cared about this."

The product should feel like a **well-designed printed object** translated to the browser — a museum exhibit caption, a fabric label on a well-made jacket, a printed monograph — rather than a SaaS web app. The dark theme should feel premium and intentional, never clinical or default.

## Aesthetic Direction

### Foundation: Dark Fintech Design System

The visual identity is documented in [`docs/portable/swiss-design-system/`](docs/portable/swiss-design-system/README.md). (The folder path is retained for link stability; the content now describes the dark fintech / warm-sunset system.) That pack is the canonical reference for tokens, components, layouts, anti-patterns, and the AI prompt template. Read it before making any visual change.

**Non-negotiable from that pack**:
- **Dark charcoal surfaces** — page `#2B2B33`; card `#1C1C22` (recesses *into* the page, darker not lighter); popover/menu `#212127`; muted/hover `#2A2A2A`. Never light fills, never `bg-white`, never pure-black page bg.
- **White ink** — primary text `#FFFFFF`; secondary `#9B9BA5`; muted/icons/placeholder `#5C5C66`. High contrast on charcoal.
- **Gold primary** — `#F5C542` with `#1C1C22` text-on-gold for AA contrast. Rare and deliberate.
- **Warm sunset gradient accent** — `.bg-gradient-sunset` / `.text-gradient-sunset` (`135deg #F56B3F → #FFD65C`), coral to amber. Reserved for hero CTAs and featured moments.
- **Status colors** — success `#3DDC97`, warning `#F7D488`, destructive `#E14B4B`, applied translucent-on-dark (token/10 fill + token/40 border + solid-color label).
- **Rounded corners** — radius tokens: `rounded-md` 12px (inputs/buttons), `rounded-lg` 16px (cards/panels), `rounded-xl` 20px (modals/featured). Never `rounded-none`.
- **Soft-glow shadows only** — `shadow-sw-xs … shadow-sw-xl`, dark blurred `rgba(0,0,0,0.3–0.5)`. Names retained from the prior system (~31 components reference them); only the values changed. Never Tailwind's `shadow-sm/md/lg`; never hard offset shadows.
- **Two-font hierarchy** — Inter (`font-sans`, body + headers; `font-serif` maps to Inter as a safety net) and Space Grotesk (`font-mono`, digits/metadata/labels). No third font.
- **Tailwind v4 configured in CSS** (`@theme inline` in `globals.css`) — there is no `tailwind.config`. Always use semantic utility classes (`bg-card`, `text-foreground`, `border-border`, …), never hardcoded hex.

### Pull toward confident premium

The dark theme is the brand — push it to feel crafted and distinctive, not generic:

- Bigger type contrast — a section header should look 2.5–3× as important as body, not 1.2×. Inter hierarchy via size + weight.
- Poster-like moments on landing/marketing surfaces: oversized hero text with a `.text-gradient-sunset` accent word, asymmetric grids, intentional charcoal whitespace as a graphic element.
- Status dots and Space Grotesk labels should feel like fintech data readouts — precise, confident, not decorative UI chrome.

### Pull toward refined dark

The dark surface can accumulate SaaS habits too. Pull toward restraint:

- Spend the gold ruthlessly. Most screens are white ink on charcoal with one moment of color. A gold button should feel rare.
- Remove any light-theme leftovers (`bg-white`, `border-black`, `text-gray-*`, light-tint alert fills, default `shadow-md`).
- Prefer typography hierarchy over decorative containers. Not everything needs a card. When it does, it's `bg-card` + `rounded-lg` + `shadow-sw-card` — soft, not heavy.
- One sunset-gradient element per screen. If the headline uses `.text-gradient-sunset`, the CTA below it is solid gold.

### Theme

**Dark only.** Dark is the brand now — the prior light theme (`#F0F0E8` canvas) was superseded by this rebrand. There is a single dark theme; no light-mode mapping and no `.dark` toggle belong in the app shell. (Resume render templates and print/PDF output remain a separate *light* document system with their own CSS — that light print path is intentional and must stay.)

### References (for the right feel)

- A premium dark dashboard with a restrained warm accent (fintech/trading tools executed with taste)
- High-end consumer-electronics interface (Bang & Olufsen, dark audio gear UI) — confident, material, calm
- A museum exhibit caption card, now rendered dark-on-dark
- A well-made fabric label or printed monograph — the "crafted object" spirit, translated to a screen

### Anti-references (what this should NOT look like)

- Generic SaaS resume builders (Resume.io, Zety — soft pastel palettes, rounded everything, decorative icons, "AI ✨" badges)
- **Stock dark-template look (Vercel/Linear/Stripe clone)** — elegant but no longer distinctive; every YC company looks like this. The dark fintech look must feel crafted and specific, not like a default dark template. Watch for: pure-black `#000` backgrounds, generic violet/indigo accents, identical card grids.
- Glassmorphism (decorative blur, glow borders), neon-on-dark, pastel cyan/purple "AI" palettes
- Hero-metric template ("3M+ resumes optimized · ⭐ 4.9 stars · 50+ countries")
- Card carousels, decorative sparklines, identical icon-headed cards, bare emoji, `✨` badges

## Design Principles

Five rules that should guide every design decision in this codebase. When in doubt, read these.

### 1. Dark, warm, confident

The surfaces are charcoal (`#2B2B33` / `#1C1C22`), the ink is white, the corners are rounded, the shadows are soft dark glows, and the accent is gold + a warm sunset gradient. This combination is the brand: confident premium structure without clinical chill or default-dark starkness. Never ship light surfaces. Never ship pure-black page backgrounds. Never ship `rounded-none` or hard offset shadows.

### 2. One primary action per region

Gold (`#F5C542`) is rare on purpose. Each logical screen region (a card, a panel, a page) should have at most one primary action — solid gold (`bg-primary`) or, for the single hero, the sunset gradient (`.bg-gradient-sunset`). Everything else is secondary, outline, or ghost. If a screen has two gold/gradient buttons, one of them is wrong.

### 3. Type does the heavy lifting

Hierarchy comes from weight and size in Inter, not from color or boxes. A section header should be 2.5–3× the body size with bold/semibold weight. A label should be small Space Grotesk (`font-mono`), sentence case (not uppercase), used for digits, metadata, and form labels. The type system carries the design — decoration is forbidden.

### 4. Asymmetric, anti-centered layouts

Left-aligned by default. Whitespace varies for hierarchy (more space above an important header, less above a continuation paragraph). Symmetric center-aligned content is the lazy default — push toward asymmetric compositions where the eye is pulled deliberately.

### 5. No AI decorative tells

The full anti-pattern list is in [`docs/portable/swiss-design-system/anti-patterns.md`](docs/portable/swiss-design-system/anti-patterns.md). The non-negotiable bans:

- **No tasteless gradient text abuse** — rainbow, neon, or decorative `background-clip: text`. NOTE: a deliberate **warm-sunset gradient accent** (`.text-gradient-sunset` on a single hero word, or `.bg-gradient-sunset` on the one hero CTA) is **on-brand and encouraged**.
- **No glassmorphism** (decorative blur, glow borders)
- **No neon-on-dark / purple-cyan "AI" palettes** — our accent is warm gold + coral/amber, never neon cyan/violet
- **No decorative side-stripe borders** (`border-left: Npx solid color` as a lazy accent on every card)
- **No decorative icons / sparkle glyphs** — only functional, mono-color
- **No bounce/elastic easing** — exponential decel only
- **No animating layout properties** — transform/opacity only
- **No nested cards**, no card carousels, no hero-metric templates
- **No bare emoji** in product UI, no `✨` badges

If any of these (except the sanctioned sunset gradient accent) appear, that's a bug.

## Constraints

- **Stack**: Next.js 16 (App Router), React 19, Tailwind CSS v4, TypeScript 5
- **System font families**: **Inter** (`font-sans`, body + headers) and **Space Grotesk** (`font-mono`, digits/metadata/labels). These two are the brand — use them.
- **Forbidden font families** (per impeccable + brand): Roboto, Open Sans, system defaults, DM Sans/Serif, Plus Jakarta Sans, Outfit, Fraunces, Newsreader, Playfair Display, Cormorant, IBM Plex *, Instrument *
- **Accessibility target**: **WCAG 2.2 AA**. 4.5:1 text contrast minimum (body text is always `#FFFFFF` on charcoal; `#5C5C66` is for placeholder/icon/metadata only, never body), full keyboard navigation, visible gold focus indicators (`ring-ring`), ARIA on interactive elements, semantic landmarks.
- **Performance budgets**: Next.js 16 App Router conventions; no barrel imports from `lucide-react`; heavy components (TipTap, dnd-kit) lazy-loaded; First Load JS under 250KB per route.
- **i18n**: en, es, zh, ja, pt currently supported — text length varies, layouts must accommodate longer translations.
- **PDF rendering**: print stylesheet exists in `globals.css`; resume preview must render identically to the printed PDF. (The print/PDF path is a separate *light* document system — its light palette is intentional and must not inherit the app's dark theme.)
